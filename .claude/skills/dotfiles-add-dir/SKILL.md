# dotfiles-add-dir

ホームディレクトリのドットディレクトリをdotfiles管理（ディレクトリごとシンボリックリンク方式）に追加する。

$ARGUMENTS = 管理対象のディレクトリパス（例: `~/.agents`）

## 前提知識

ディレクトリごとシンボリックリンク方式では、`dot_foo` ディレクトリが `~/.foo` そのものになる。つまり：
- **git管理しないファイルも含めて、元ディレクトリの中身すべてが `dot_foo/` に存在する必要がある**
- `.gitignore` の許可リスト方式（`*` で全除外 → `!` で管理対象のみ許可）でgit追跡対象を制御する

## 手順

### 1. 元ディレクトリの中身を確認

`$ARGUMENTS` の中身を一覧表示し、ユーザーに以下を確認する：
- git管理対象にするファイル/ディレクトリはどれか

### 2. dot_ディレクトリ作成

元ディレクトリの中身をまるごとコピーする（rsync等を使用）。

```
rsync -a ~/.foo/ dot_foo/
```

### 3. .gitignore作成

`dot_foo/.gitignore` を許可リスト方式で作成する。`dot_claude/.gitignore` を参考にすること。

```
*
!.gitignore
!管理対象ファイル1
!管理対象ファイル2
```

サブディレクトリ内にgit管理しないファイルがある場合は、そのサブディレクトリにも `.gitignore` を作成する。

### 4. install.sh更新

`install.sh` の「ディレクトリごとシンボリックリンク」セクションに追加する。

```
ln -sfn "$(pwd)/dot_foo" ~/.foo
```

### 5. 報告

変更内容をまとめて報告する（作成したファイル、git管理対象、gitignore対象）。
